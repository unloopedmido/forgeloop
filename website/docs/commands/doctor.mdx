## Usage

```bash
forgeloop doctor [--dir ./project]
```

Loads the ForgeLoop config from `--dir` (default: current directory), then checks paths derived from the manifest:

- Config file path that was loaded
- `srcDir`, `configDir`
- `commandsDir` / `eventsDir` when preset uses handlers
- `coreDir` for **advanced**
- `.gitignore` when `features.git`
- `eslint.config.js` + `.prettierrc.json` when tooling is **eslint-prettier**
- `biome.json` when tooling is **biome**
- Prisma paths when database enabled: `prisma.config.ts`, `prisma/schema.prisma`, and client file under `src/core/...` (advanced) or `src/lib/...` (modular)
- `Dockerfile` when Docker enabled
- `.github/workflows/ci.yml` when CI enabled
- `.env` presence and required values:
  - always: `DISCORD_TOKEN`, `CLIENT_ID`, `GUILD_ID`
  - database projects: `DATABASE_URL`

Each missing path or missing env value increments issues and sets exit code **1**. If no issues → **`✓ Project looks healthy.`**
