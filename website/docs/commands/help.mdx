Running `forgeloop`, `forgeloop help`, or `forgeloop --help` prints the same summary as implemented in the CLI: hero text, command lines, init wizard note, and feature flags (`--database`, `--orm`, `--tooling`, `--git`, `--docker`, `--ci`, `--install`, `--dry-run`, `--yes`).

Global aliases:

- `-h` → `--help`
- `-V` → `--version`
- `-y` → `--yes`
- `-d <path>` → `--dir <path>`

Command-specific help:

- `forgeloop init --help`
- `forgeloop add --help`
- `forgeloop deploy --help`
- `forgeloop doctor --help`
- `forgeloop info --help`

## Command list (quick reference)

| Command | Purpose |
|---------|---------|
| `init` | Scaffold a project |
| `add command` / `add event` | Generate handler files |
| `deploy commands` | Push slash commands to Discord |
| `doctor` | File presence checks |
| `info` | Manifest summary |
| `version` | Print CLI package version |

For full detail, use the sidebar **Command reference** section or start at [`init`](./init.mdx).
